awsdo bastion - Start a port forwarding session through a bastion host

USAGE:
    awsdo bastion [--profile <aws cli profile>] [--name <bastion name>]
                    [--instance <bastion instance id>] [--host <remote host>]
                    [--port <remote port>] [--local <local port>]

DESCRIPTION:
    Creates a port forwarding tunnel through a bastion host using AWS SSM.
    All parameters are optional if a bastion is configured. If --name is
    not specified and multiple bastions exist, the default bastion is used.
    If local port is not specified, an available port is auto-assigned.

    When using --name without specifying a profile:
    - The tool first searches for the bastion in the default profile
    - If not found, it searches all other profiles
    - Once found, it uses the bastion's associated profile to launch the session

    If both --name and --profile are specified, the tool only searches
    for the bastion in the specified profile.

OPTIONS:
    --profile, -p         AWS CLI profile to use
    --name               Name of the configured bastion to use
    --instance           Bastion EC2 instance ID (overrides configured instance)
    --host               Remote host to forward to (overrides configured host)
    --port               Remote port to forward (overrides configured port)
    --local              Local port to bind to (overrides configured local port)

EXAMPLES:
    awsdo bastion
        Uses the default bastion on the default AWS profile.

    awsdo bastion -p dev
        Uses the default bastion on the specified profile.

    awsdo bastion --name my-prod-db
        Searches for bastion named "my-prod-db" across all profiles.

    awsdo bastion -p dev --name my-db
        Uses the bastion named "my-db" from the dev profile only.
